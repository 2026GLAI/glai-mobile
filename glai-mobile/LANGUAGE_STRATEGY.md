LANGUAGE_STRATEGY — Language Handling & Switching Rules
Purpose

This document defines how GLAi handles language clearly, respectfully, and safely.

Language is not a feature.
Language is trust.

Incorrect language behavior breaks credibility instantly.

Core Principle

Greek (Ελληνικά) is the primary and default language of GLAi.

All other languages exist to support understanding —
never to replace or diminish Greek.

Default Language Rule

If the user is in Greece → Greek is preferred

If the user language is unclear → Greek is used

If a mistake occurs → Greek is the safe fallback

Greek is never hidden or deprioritized.

Language Detection Strategy

GLAi determines language using:

Browser / device language

User input language

Regional context (when available)

Detection is best-effort, never absolute.

Error Handling in Language Detection

If language detection is uncertain:

GLAi responds in Greek

A simple, optional language switch may be offered

No apology for “wrong language”

No automatic switching mid-conversation

Stability is more important than correctness.

Language Switching Rules

Language switching must be explicit or clearly implied

GLAi must never switch languages mid-sentence

GLAi must never mix languages in one response

One response = one language

If a switch occurs, it applies from the next response only.

Tourist & Non-Greek Users

For non-Greek users:

GLAi uses simple, neutral language

Avoids local slang or cultural shorthand

Keeps sentences short

Avoids bureaucratic phrasing

The goal is understanding, not fluency.

Legal & Official Context

In legal, tax, or official matters:

Greek terminology takes priority

Translations are supportive, not authoritative

Original Greek terms may be shown when clarity requires it

GLAi must never replace official Greek legal wording with interpretation.

Forbidden Language Behaviors

GLAi must never:

Display broken encoding or symbols

Show untranslated system strings

Mix languages for style or effect

Assume language preference permanently

Force language choice at entry

Language must feel natural, not managed.

Persistence Rule

Language preference is session-level only.

No long-term language memory is assumed unless explicitly implemented later.

Clean starts preserve trust.

Final Rule

If language handling draws attention to itself,
it has already failed.

Correct language behavior feels invisible.

End of Document
